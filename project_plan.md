# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 친환경 바이오 플라스틱 조건별 생분해 기간 예측 및 환경 영향 평가기

예) 기분과 시간에 따른 활동 추천기

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

예) 사용자가 현재 기분과 사용할 수 있는 시간을 입력하면, 조건에 맞는 활동을 추천해 주고 싶다.

내 설명: 최근 환경 오염 대안으로 바이오 플라스틱이 주목받고 있으나, 이는 종류와 매립되는 환경(토양, 바다, 퇴비화 시설) 및 외부 조건(온도)에 따라 분해 기간이 달라진다. 소비자가 올바른 분해 조건을 모른 채 무분별하게 배출하면 친환경 효과가 떨어진다. 따라서 사용자가 플라스틱 종류와 매립 환경, 온도 조건을 입력하면 정확한 예상 생분해 기간을 계산해 주고, 올바른 배출 팁과 탄소 절감 효과를 안내하여 친환경적인 선택을 돕는 프로그램을 만들고자 한다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 플라스틱 종류: 조회하고 싶은 바이오 플라스틱 이름 또는 번호 선택 (PLA, PHA, PBS 등)
- 매립 및 배출 환경: 플라스틱이 버려질 환경 선택 (1. 토양, 2. 바다, 3. 퇴비화 시설)
- 환경의 평균 온도 상태: 분해 장소의 온도 조건 (1. 저온/겨울, 2. 적정 온도, 3. 고온/여름)

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 데이터 매칭: 2차원 리스트를 반복문(for)으로 순회하며 사용자가 입력한 플라스틱 종류가 존재하는지 확인하고, 선택한 환경에 맞는 '기본 분해 일수' 데이터를 추출한다.
- 가중치 연산 알고리즘: 온도 조건에 따라 조건문(if-elif-else)을 활용해 가중치 연산을 수행합니다. (저온: 기본 일수 $\times 1.5$, 고온: 기본 일수 $\times 0.8$, 적정온도: 기본 일수 $\times 1.0$)
- 예외 처리: 등록되지 않은 소재를 입력하거나 메뉴 번호 범위를 벗어난 값을 입력하면 오류 메시지를 출력하고 재입력을 유도합니다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 예측 결과: 온도 가중치가 반영되어 최종 계산된 생분해 예상 기간 (일 단위를 개월 또는 년 단위로 보기 좋게 환산하여 출력)
- 환경 정보: 해당 신소재의 올바른 배출 팁 및 기존 석유 기반 플라스틱 대비 '탄소 배출 절감률(%)' 성적표 출력

## 6. 사용할 제어구조

### 조건문을 사용할 부분

예) 사용자의 예산보다 가격이 낮은 항목만 추천한다.

내 계획: 사용자가 선택한 매립 환경 번호(1, 2, 3)에 따라 2차원 리스트의 몇 번째 열(1번, 2번, 3번 열)에서 기본 데이터를 가져올지 분기할 때 사용합니다.

입력받은 온도 조건에 따라 가중치 배율(1.5, 1.0, 0.8)을 결정하는 수식 연산 분기에 사용합니다.

데이터 검색 결과가 비어있는(None) 경우의 예외 처리에 사용합니다.


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획: 메뉴 1번 선택 시, 2차원 리스트의 모든 행을 하나씩 돌면서 현재 프로그램에 등록된 바이오 플라스틱의 이름과 탄소 절감 정보를 리스트 형식으로 화면에 나열할 때 사용합니다.

메뉴 2번 검색 시, 사용자가 입력한 플라스틱 이름이 2차원 리스트에 존재하는지 첫 번째 열(row[0])을 한 줄씩 검사할 때 사용합니다.

프로그램이 종료(3번)를 선택하기 전까지 계속해서 전체 시스템이 유지되도록 while True 무한 루프를 제어할 때 사용합니다.
