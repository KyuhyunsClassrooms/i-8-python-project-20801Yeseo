# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 친환경 바이오 플라스틱 조건별 생분해 기간 예측

이 프로그램을 만든 이유:
최근 환경 오염의 대안으로 다양한 바이오 플라스틱 신소재가 주목받고 있습니다. 그러나 이들은 소재의 종류뿐만 아니라 매립되는 환경(토양, 바다, 시설)과 외부 온도에 따라 썩는 기간이 수백 일 이상 차이가 납니다. 소비자들이 이를 알지 못하고 무분별하게 배출하면 친환경 효과가 반감됩니다. 이에 소비자가 플라스틱 종류와 환경 조건을 입력하면 정확한 예상 생분해 기간을 연산해 주고, 올바른 배출 지침과 탄소 절감 성적을 제공하여 주도적인 친환경 환경 보호 실천을 유도하고자 제작했습니다.

---

## 2. 문제 분해 및 모델링

### 입력
소재명: 조회할 바이오 플라스틱 이름 (예: PLA, PHA, PBS)
매립 환경: 플라스틱이 배출 및 버려질 장소 번호 (1. 토양 / 2. 바다 / 3. 퇴비화 시설)
환경 온도: 해당 장소의 평균 온도 조건 번호 (1. 저온/겨울 / 2. 적정 온도 / 3. 고온/여름)

### 처리
데이터 검색: 2차원 리스트 내부를 순회하며 사용자가 입력한 키워드와 일치하는 플라스틱 데이터 행(Row) 추출
가중치 수식 연산: 입력받은 온도 조건에 따라 조건문 분기를 거쳐 기본 분해 일수에 가중치(1.5배, 1.0배, 0.8배)를 곱함
시간 단위 환산: 계산된 최종 일수(day)를 몫(//)과 나머지(%) 연산자를 이용해 '년'과 '일' 단위로 자동 변환
오류 예외 처리: 데이터가 없거나 잘못된 번호 입력 시 오류 문구를 출력하고 시스템을 유지함

### 출력
종합 시뮬레이션 리포트: 가중치가 반영되어 최종 계산된 생분해 예상 기간 (일 단위 및 년/일 단위 환산 문장)
환경 성적표: 기존 석유 플라스틱 대비 해당 소재가 절감한 탄소 배출 비율(%) 및 친환경 배출 지침 팁 출력

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: plastic_data

한 행의 의미: 하나의 특정 바이오 플라스틱 신소재가 가진 고유 명칭, 환경별 생분해 기간 데이터, 배출 지침 및 탄소 절감 정보

각 열의 의미: 

| 열 번호 | 의미 | 실제 예시 |
| 0번 열 | 바이오 플라스틱 소재 명칭 (문자열) | "PLA (옥수수 전분 계열)" |
| 1번 열 | 토양 매립 시 기본 생분해 기간 (정수, 단위: 일) | 365 |
| 2번 열 | 해양(바다) 투기 시 기본 생분해 기간 (정수, 단위: 일) | 730 |
| 3번 열 | 전문 퇴비화 시설 배치 시 기본 생분해 기간 (정수, 단위: 일) | 90 |
| 4번 열 | 소비자를 위한 올바른 친환경 분해 배출 팁 (문자열) | "반드시 전문 퇴비화 시설로 보내야 해요!" |
| 5번 열 | 기존 석유 플라스틱 대비 탄소 배출 절감 비율 (정수, 단위: %) | 70 |

이 2차원 리스트를 사용한 이유:
바이오 플라스틱은 소재 종류라는 '행'의 기준과, 매립 환경이라는 '열'의 기준이 결합하여 하나의 고유한 분해 기간 일수 데이터를 형성합니다. 인덱스 번호를 통해 특정 데이터를 골라내기 위해 2차원 리스트를 사용했습니다.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|show_menu()|메인 화면에 사용자가 선택할 메뉴 번호판을 출력함|없음|없음 (콘솔 화면 텍스트 출력)|
| find_plastic_row(data, name) | 2차원 리스트에서 사용자가 입력한 글자가 포함된 플라스틱 행 데이터(리스트)를 검색함 | data(2차원 리스트), name(문자열) | 찾은 경우 일치하는 한 행의 리스트 반환, 없으면 None 반환 |
| calculate_final_days(base_days, temp_choice) | 선택한 환경의 기본 일수와 온도 번호를 받아 가중치를 반영한 최종 분해 일수를 연산함 | base_days(정수), temp_choice(문자열) | 가중치가 계산된 최종 분해 일수(정수) 반환 |
| print_report(name, final_days, tip, carbon_rate) | 계산된 일수를 판단하여 365일 이상이면 년/일로 변환하고 탄소 절감률과 팁을 종합 리포트로 출력함 | name, final_days, tip, carbon_rate | 최종 예측 리포트 텍스트 화면 출력 |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름: find_plastic_row(data, name)

설명:
프로그램의 데이터베이스 역할을 하는 2차원 리스트(data)에서 사용자가 입력한 텍스트(name)를 기반으로 핵심 행을 찾아내는 역할을 합니다. for row in data:를 통해 표의 첫 번째 줄부터 순서대로 순회하며, 만약 사용자가 입력한 문자열이 해당 행의 0번 열(이름 칸)에 포함되어 있다면(if name in row[0]:), 그 플라스틱의 모든 데이터가 담긴 행 리스트(row)를 통째로 반환(return)합니다. 만약 표를 끝까지 다 뒤졌는데도 일치하는 정보가 없다면 None을 반환하여 메인 제어부에서 예외 처리를 할 수 있도록 돕습니다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가? 
사용자가 메인 메뉴에서 고른 번호(1, 2, 3)에 따라 서로 다른 기능이 실행되도록 분기할 때 사용했습니다.
입력된 배출 환경 번호(1, 2, 3)에 따라 2차원 리스트 내의 특정 열 인덱스(1번, 2번, 3번 열)를 선택하여 base_days에 대입할 때 활용했습니다.
온도 조건 번호(1, 3)를 검사하여 가중치 수식(* 1.5, * 0.8)을 다르게 적용할 때 사용했습니다.



### 반복문 사용 부분

무엇을 반복 처리했는가?
프로그램이 종료 번호(3번)를 받기 전까지 전체 시스템이 꺼지지 않고 계속 구동되도록 while True: 무한 루프를 제어할 때 사용했습니다.
메뉴 1번 선택 시, for row in plastic_data:를 통해 등록된 모든 바이오 플라스틱 정보를 한 줄씩 꺼내어 나열식 목록으로 출력할 때 사용했습니다.

### 조건문과 반복문이 함께 작동한 부분
find_plastic_row 함수 내부에서 for row in data:라는 반복문이 실행되는 도중, 그 내부에 if name in row[0]:이라는 조건문을 중첩하여 삽입했습니다. 이 구조를 통해 반복문이 표를 한 줄씩 훑어 내려가다가 사용자가 원하는 조건과 일치하는 순간을 조건문으로 포착하여 연산을 수행하는 복합 제어 구조를 달성했습니다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력: 플라스틱명: "PHA" / 환경: 3 (퇴비화 시설) / 온도: 3 (고온)

예상 결과: PHA의 퇴비화 기본 30일 $\times$ 고온 가중치 0.8배 = 최종 24일 출력 및 PHA 관련 팁과 주의 문구 출력.

실제 결과: 예상대로 최종 예상 기간 약 24일로 정확히 연산되어 관련 팁과 주의 문구가 함께 출력됨.


### 테스트 2: 예외 또는 경계 사례

입력: 플라스틱명: "일반 페트병" (등록되지 않은 소재 입력)

예상 결과: 2차원 리스트 탐색 후 결과가 없으므로 None을 반환하고 "❌ 등록되지 않은 플라스틱 이름입니다..." 안내문 출력 후 메뉴로 복귀.

실제 결과: 프로그램이 다운되거나 강제 종료되지 않고, 정확히 지정한 예외 처리 경고문이 출력된 후 상위 메뉴판으로 안전하게 복귀함.


### 발견한 오류와 수정한 방법
오류 상황: 초기 테스트 시 온도 가중치를 계산하는 과정에서 실수형(Float) 연산이 일어나 최종 소요 기간이 약 547.5일과 같이 소수점으로 지저분하게 출력되는 문제가 발생했습니다.

수정 방법: calculate_final_days 함수의 반환값 수식 전체를 파이썬의 정수 변환 함수인 int()로 감싸주어(return int(base_days * 1.5)), 소수점 이하 자리를 깔끔하게 버리고 정확한 정수 일수 데이터만 표현되도록 수정하여 해결했습니다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분:
수행평가 만점 기준을 충족할 수 있도록 바이오 플라스틱 종류와 매립 환경을 행과 열로 매칭하는 2차원 리스트 구조를 고안하는 데 도움을 받았습니다. 또한, 코드의 가독성을 높이기 위해 메인 로직에서 메뉴판, 탐색, 연산, 출력을 담당하는 총 4개의 독립된 함수로 깔끔하게 쪼개어 연결하는 함수 분리 설계 지침을 지원받았습니다.

Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분:
AI는 처음에 플라스틱 이름과 환경별 일수, 배출 지침이라는 1차원적인 단순 조회용 데이터 구조만 제안했습니다. 하지만 이를 그대로 사용하지 않고 신소재의 환경적 가치를 다각도로 평가하기 위해 '탄소 배출 절감률(%)'이라는 5번 열 데이터를 2차원 리스트에 스스로 추가하여 결과에 함께 출력하도록 수정했습니다.

최종 코드에서 내가 직접 판단하거나 수정한 부분:
단순히 계산된 '총 일수' 숫자만 보여주는 것은 사용자가 직관적으로 기간을 체감하기 어렵다고 판단했습니다. 이에 보고서 출력 함수 내부에 파이썬의 몫 연산자(//)와 나머지 연산자(%)를 직접 결합한 판정문 구조를 도입하여, 365일 이상일 때 자동으로 'X년 X일' 형태의 문장으로 자동 변환해 주는 것을 구현했습니다. 또한 바이오 플라스틱 생분해 기간이 조건에 따라 크게 달라지는 것을 고려해 마지막 계산 결과를 보여줄 때 사용자가 특정 수치로만 생각하지 않을 수 있게 문구를 추가했습니다.

## 9. 한계와 개선 방안

현재 프로그램의 한계:
현재는 하드코딩된 2차원 리스트 내부의 3가지 플라스틱(PLA, PHA, PBS) 데이터만 조회할 수 있어서, 새로운 바이오 플라스틱 신소재가 개발되거나 사용자가 새로운 항목을 추가하고 싶을 때 프로그램 실행 중에 실시간으로 데이터를 추가하는 기능이 부족합니다.

다시 개선한다면 추가하고 싶은 기능:
메인 메뉴에 '4. 새로운 바이오 플라스틱 등록하기' 기능을 추가하고 싶습니다. 사용자로부터 이름, 토양/바다/시설 분해 일수, 탄소 절감률을 입력받아 기존 plastic_data 리스트에 행을 추가하는 데이터 동적 확장 기능을 보완하고 싶습니다.

## 10. 코드인터뷰 준비

다음 질문에 답할 수 있도록 준비하세요.

1. 내 프로그램의 입력-처리-출력은 무엇인가?
- 입력은 조회할 플라스틱 이름, 매립 환경 번호, 온도 조건 번호입니다.
처리는 2차원 리스트 순회 검색, 온도에 따른 가중치 수식 연산, 일수를 년/일 단위로 바꾸는 환산 처리입니다.
출력은 최종 연산된 일수와 년/일 단위 문장, 탄소 배출 절감률 성적표와 올바른 배출 팁 문장입니다.
2. 2차원 리스트에서 한 행과 각 열은 무엇을 의미하는가?
- 한 행(Row)은 하나의 바이오 플라스틱 신소재가 가진 종합 환경 데이터 세트를 의미합니다.
각 열(Column)은 0번 열부터 순서대로 소재 이름, 토양 분해 기간, 바다 분해 기간, 퇴비화 시설 분해 기간, 올바른 배출 팁, 그리고 석유 대비 탄소 절감 비율을 의미합니다.
3. 가장 중요한 함수는 무엇이고 어떤 역할을 하는가?
- find_plastic_row(data, name) 함수입니다. 반복문(for)으로 2차원 리스트 데이터베이스를 한 줄씩 순회하고 조건문(if)으로 사용자가 입력한 키워드를 대조하여, 일치하는 특정 플라스틱의 행 데이터를 통째로 안전하게 추출해 오는 핵심 필터링 역할을 합니다.
4. 조건문과 반복문은 어디에서 사용되었는가?
- 반복문은 프로그램 전체를 유지하는 메인 while True 루프와 메뉴 1번에서 전체 데이터를 한 줄씩 꺼낼 때 사용되었습니다.
조건문은 사용자 메뉴 선택 분기, 선택한 환경별 열 데이터 매칭, 온도별 가중치 연산 분기에서 사용되었습니다.
복합 구조로는 find_plastic_row 함수 내부에 for 반복문 안에 if 조건문이 중첩되어 작동합니다.
5. Gemini가 도와준 부분과 내가 직접 수정한 부분은 무엇인가?
- Gemini는 2차원 리스트의 기초 뼈대와 4개 함수로 기능을 분리하는 구조적 설계를 도와주었습니다.
내가 수정한 부분은 2차원 리스트에 '탄소 배출 절감률' 열 데이터를 직접 추가해 확장한 점, 몫(//)과 나머지(%) 연산자를 직접 적용해 'X년 X일' 환산 문구 알고리즘을 구현한 점, 잘못 입력했을 때 프로그램이 안 꺼지도록 예외 처리 경고 흐름을 강화한 점입니다.
